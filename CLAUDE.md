# mm 專案指示

## Guideline 索引

本專案的工作規範。動工前先確認相關條目：

- **溝通語言** — 給使用者的說明/建議一律繁體中文。見下「溝通語言」。
- **UI 版面** — 一律依視窗比例、解析度無關、不寫死像素。見下「UI 版面」。
- **美術風格** — 美式 D&D × CRPG 寫實 × 英雄高奇幻 × 英雄寫實比例；人物/NPC/怪物一致。見下「美術風格」與 `docs/art-style-guide.md`。
- **資產處理（容量）** — 圖片進 repo 前一律壓 WebP、不放原圖（場景 q82@1280／肖像 q88@1024／去背 sprite q85 保 alpha）；PBR 材質與 normal map 例外不 lossy。見 `docs/art-style-guide.md`「資產處理」。
- **不需向後相容** — 現階段為 pre-release 開發，breaking change 一律可接受，不為相容/存檔遷移付出任何成本。見下「不需向後相容」。
- **開發工具（tools/）** — 離線跑的工具索引見 `tools/README.md`。含 `gen_parchment.gd`（程序化生成羊皮 UI 貼圖，可指定尺寸）、戰鬥/升級模擬器、任務驗證器等。

## 不需向後相容：現階段所有 breaking change 都可接受

本專案目前是 **pre-release 開發期**，沒有正式使用者、沒有需要保護的正式存檔。因此**完全不需要為向後相容或存檔相容付出任何成本**：

- **不寫相容/遷移程式碼**：不要寫「相容 v1–v4」「載到舊格式則 fallback」「保留舊欄位以防舊存檔」這類版本協商或回退邏輯。要改格式就直接改。
- **存檔格式可隨意改、不需升版號保相容**：改 save schema、地圖 JSON、資料格式、函式簽章時，直接改成最乾淨的樣子，**一併更新所有呼叫端與既有 content/test 資料**即可。舊存檔壞掉沒關係。
- **優先選乾淨重設計，而非相容層**：當「改乾淨」與「加相容層」二選一時，一律選改乾淨。
- 設計文件/spec 也不要把「避免存檔升級」當成設計驅動因素——該升就升、該砍就砍。

理由：避免在無人使用的早期專案堆積相容包袱與防禦性程式碼，保持架構乾淨好改。等專案真正對外、有正式存檔時再另立相容策略。

## 溝通語言：給使用者的建議一律用繁體中文

在這個專案，凡是「要對使用者說的建議／推薦／方案比較／利弊權衡／下一步提議」一律用**繁體中文**書寫。例如：技術選型建議、做法推薦、素材／工具來源建議、架構取捨說明、要不要做某件事的提議。

此規則只規範「對使用者的說明與建議」的語言。不影響：程式碼、檔案內容、commit 訊息、以及 sub-agent 的工作指令——那些維持既有慣例即可。

## UI 版面：一律依視窗比例（解析度無關），不寫死像素

所有 HUD／UI 版面**一律依視窗比例**定位與決定大小，**不得寫死像素**寬高或座標。目標：任何視窗大小、含全螢幕都正確、不會擠成一角或變太小。具體做法：

- **定位與寬度用 anchor 比例**：例如「靠左占畫面下方約 70% 寬、右側留白」→ `anchor_left = 0.0` / `anchor_right = 0.70`（而非固定 900px）。貼邊／向某方向成長用 `grow_*` + 比例 anchor，不要用固定 offset 當主要尺寸來源（小幅邊距 offset 可接受）。
- **子元素平均分攤用 `size_flags`**：一排同類元素（如 6 張隊友卡）讓每個 `size_flags_horizontal = SIZE_EXPAND_FILL`，由容器平均分配寬度。
- **條狀／比例元素用 anchor 比例，不用固定像素**：如 HP/MP 條的填色用 `fill.anchor_right = ratio`（隨父寬自動縮放），不要 `fill.size = ratio * 寫死寬`。
- 字級、間距等可給固定值，但**版面骨架的寬高定位必須是比例式**。
- 既有元件若用寫死像素佈版（早期 placeholder），在動到它時順手改成比例式。

理由：先前隊伍 HUD 用固定 900px，在較大／全螢幕視窗會顯得太小且擠在左側；改成 anchor 比例後在各解析度都正確。

## 美術風格：美式 D&D（CRPG 寫實），人物/NPC/怪物一致

所有 **2D 角色美術**（英雄、NPC、怪物）一律遵循 **`docs/art-style-guide.md`**。摘要：

- **定位**：美式 D&D × **CRPG 寫實（Baldur's Gate 3 / Solasta 風）** × **英雄高奇幻**氛圍（明亮飽和暖調）× **英雄寫實**比例（寫實五官+略英雄化、臉清楚）。
- **一致性錨點**：半寫實渲染（無卡通描邊）、固定左上暖色主光、共用暖調色盤、主體與背景分離、無內建文字/邊框。
- **生圖**：用文件內的**固定風格前綴 + 每角色 Subject + negative prompt**（換臉狀態變體加表情字尾），確保整套一致。
- 3D 場景維持寫實 PBR；2D 角色需與其寫實度/色溫和諧。

新增/委託任何角色圖前，先讀 `docs/art-style-guide.md` 並沿用其 prompt 配方。
